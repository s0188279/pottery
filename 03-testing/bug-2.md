# Баг-репорт 2: Отсутствие уменьшения свободных мест при успешной брони

### 1. Симптом / Описание проблемы
Согласно бизнес-инвариантам из `data-model (1).md`, количество свободных мест на слоте должно уменьшаться после того, как клиент совершил успешную запись. Изначально на экране подтверждения места не уменьшались, что в реальной системе привело бы к овербукингу.

### 2. Промпты к ИИ для исправления
> "В функции подтверждения бронирования confirmBooking() напиши логику, которая перед переходом на экран успеха находит выбранный слот в массиве MOCK_SLOTS и вычитает из его параметра free_seats единицу."

### 3. Исправление
В функцию `confirmBooking` добавлена мутация локального состояния:
```javascript
if (state.selectedSlot.free_seats > 0) {
    state.selectedSlot.free_seats -= 1;
}
```
