# Баг-репорт 1: Отсутствие валидации номера телефона при авторизации

### 1. Симптом / Описание проблемы
На экране авторизации пользователь мог оставить поле ввода телефона пустым, ввести любой четырехзначный код "1111" и успешно пройти на экран расписания. Это нарушает критерии приемки US-001 (Валидация формата номера телефона).

### 2. Промпты к ИИ для исправления
> "В функции handleLogin() добавь проверку на заполненность поля телефона. Если поле пустое или длина меньше 10 символов, выводи предупреждение alert и прерывай выполнение функции, не давая зайти с кодом 1111."

### 3. Исправление и ручная проверка
В код функции `handleLogin()` добавлено условие:
```javascript
if(!phone || phone.length < 10) {
    alert("Ошибка: Введите корректный номер телефона!");
    return;
}
```
При ручной проверке с пустым телефоном вход блокируется. Ошибка устранена.
