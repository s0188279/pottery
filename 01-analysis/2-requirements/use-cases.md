# Сценарии использования (Use Cases)

## UC-1: Запись на мастер-класс с прокатом инвентаря

* **Актер:** Авторизованный клиент.
* **Предусловия:** Пользователь находится на экране списка слотов (`SCR-002-slot-list`)[cite: 1]. На бэкенде есть слоты со свободными местами[cite: 1].

### Основной сценарий (Успешный путь)
1. Пользователь выбирает интересующий слот мастер-класса и нажимает кнопку «Записаться»[cite: 1].
2. Приложение открывает экран оформления бронирования (`SCR-004-booking`), отображая детали слота и чекбокс «Нужен прокат инструментов и фартука»[cite: 1].
3. Пользователь активирует чекбокс проката и нажимает «Подтвердить запись»[cite: 1].
4. Приложение отправляет POST-запрос на создание бронирования `POST /api/v1/bookings` с флагом `needs_rent: true`[cite: 1].
5. Бэкенд атомарно проверяет наличие мест, создает бронь и возвращает статус `201 Created`[cite: 1].
6. Приложение перенаправляет пользователя на экран успешного бронирования (`BS-002-booking-success`)[cite: 1].

### Альтернативные сценарии
* **1а. Места на бэкенде закончились в момент оформления:**
  * 5. Бэкенд возвращает ошибку `409 Conflict` (из-за отсутствия свободных мест)[cite: 1].
  * 6. Приложение показывает пользователю уведомление: *«Извините, места на этот мастер-класс только что закончились»* и возвращает на экран расписания[cite: 1].

## UC-2: Отмена бронирования клиентом

* **Актер:** Авторизованный клиент.
* **Предусловия:** Пользователь находится на экране деталей своего бронирования (`SCR-006-booking-details`), бронь в статусе «Подтверждено»[cite: 1].

### Основной сценарий (Успешный путь)
1. Приложение проверяет разницу между текущим временем устройства и временем начала слота. Разница составляет более 2 часов[cite: 1].
2. Кнопка «Отменить запись» отображается как активная[cite: 1].
3. Пользователь нажимает «Отменить запись»[cite: 1].
4. Приложение показывает модальное окно подтверждения отмены (`BS-003-cancel-confirm`)[cite: 1].
5. Пользователь подтверждает действие[cite: 1].
6. Приложение отправляет POST/PATCH запрос `POST /api/v1/bookings/{id}/cancel`[cite: 1].
7. Бэкенд обновляет статус брони на «Отменено», освобождает место на слоте и возвращает `200 OK`[cite: 1].
8. Приложение обновляет экран деталей, переводя бронь в статус «Отменено клиентом», кнопка отмены скрывается[cite: 1].

### Альтернативные сценарии
* **2а. До начала занятия осталось менее 2 часов:**
  * 1. Приложение определяет, что до старта слота осталось менее 2 часов[cite: 1].
  * 2. Кнопка «Отменить запись» переводится в состояние `disabled` (заблокирована)[cite: 1].
  * 3. Рядом с кнопкой отображается информационный текст: *«Отмена невозможна менее чем за 2 часа до начала занятия»*[cite: 1].
