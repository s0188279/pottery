# Фича: Авторизация клиента по номеру телефона

### 1. Цель
Реализовать стартовый экран входа в приложение для идентификации пользователя, чтобы предоставить ему доступ к расписанию и бронированиям.

### 2. Требования (На основе US-001)
- Поле ввода номера телефона с валидацией формата.
- Кнопка «Получить код».
- Поле ввода 4-значного OTP-кода (симуляция).
- Переход на главный экран при вводе кода "1111".

### 3. Отправленные промпты ИИ
**Промпт 1 (Генерация UI):**
> "Напиши простой фронтенд-компонент экрана авторизации для мобильного приложения гончарной мастерской. Нужна валидация телефона, поле для ввода 4 цифр OTP-кода. При вводе '1111' должен происходить успешный вход и переключение состояния на авторизованного пользователя. Сделай код чистым, на [Укажите ваш стек, например: React Native / Flutter / HTML+JS]."

### 4. Реализация
Создан компонент AuthScreen. Логика проверки OTP вынесена в локальный стейт, при успешном вводе токен авторизации сохраняется условно в сессии (согласно api-sequence (1).md).
