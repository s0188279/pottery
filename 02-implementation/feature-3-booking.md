# Фича: Оформление бронирования с прокатом инвентаря

### 1. Цель
Дать возможность клиенту забронировать конкретный слот из расписания и выбрать, нужны ли ему инструменты и фартук напрокат.

### 2. Требования (На основе US-005 и UC-1)
- Экран оформления бронирования, открывающийся по клику на слот.
- Чекбокс/переключатель «Нужен прокат инструментов и фартука».
- Кнопка «Подтвердить запись», отправляющая данные.
- Экран/сообщение об успешном бронировании.

### 3. Отправленные промпты ИИ
**Промпт 1 (Генерация UI бронирования):**
> "Напиши форму бронирования BookingScreen. Форма принимает ID выбранного слота, содержит чекбокс 'Нужен прокат инструментов и фартука'. По нажатию на кнопку 'Подтвердить запись' симулируй отправку POST /api/v1/bookings и при успешном ответе (201 Created) переводи UI на экран успеха, как описано в api-sequence (1).md."

### 4. Реализация
Интегрирован чекбокс `needs_rent`. По клику на «Подтвердить» состояние брони переходит в `created`, а у слота динамически уменьшается количество свободных мест на 1 (инвариант из data-model (1).md).
